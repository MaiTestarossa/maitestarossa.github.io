序

最近研究了一堆小主机和软路由，今年想完成简单升级的目标。

目前环境

  1. DELL R230 噪音太高，噪音改造后还面临待机功耗40w起步的问题，实在浪费电目前又没什么需求。
  2. N1 armbian 拿来放openwrt docker，更新之类怕挂，又懒得折腾，但openwrt和openclash都算能跑的稳，升级意义有限，但早晚要换。
  3. QNAP 9盘位，这种系统当Linux用比较尴尬，重启会清除配置，必须搞开机预处理脚本，主要用于备份、存文件以及一些下载组件等，部分轻量工作的docker也交给它。
  4. 3代cpu的笔记本，待机10w，用来放qq、微信等国产垃圾软件环境，所有聊天等操作都通过RDP连到这台机器。这台机器还兼顾跳板，装了可视化的wol工具，用来在外网访问时唤醒其他家庭网络机器。
  5. ROS网关，主要架设wiregurad中心节点，公司cn2 gia的网络本来也打算放上面并配置路由，无奈v7版ros不会玩，v6的教程似乎套不上去，l2tp ipsec配置了半天也没法让本地机走。
  6. apple tv，可惜appletv不能装surge之类的软件，不然当作旁路网关多好（但这东西平时是睡眠的，仔细一想并不合适当网关）。

需求

  1. 一套完整的Linux环境长期运行机用于折腾和docker(可能适合放在aio机里？)
  2. adguard，目前放qnap里，qnap重启时间实在太长，这段时间里所有用adguard作为dns的机器都等于瘫痪了。
  3. ubnt控制器，目前放qnap上，没用docker，用的第三方包，没啥问题，但靠第三方维护包更新慢，多一道分发多一道风险，qnap维护重启时也面临ap没ac可用的情况。
  4. 开发环境，想要一台机器专注用写代码，因为现在工作是直接远程到日本，全部企业级工具，为了不让自己丧失流行代码编写能力，有必要平时玩玩。
  5. openclash新环境，可以考虑直接买个mac mini用surge当网关，但实际上openclash也足够能用了。

目标

  1. 一台迷你主机用来装开发环境，后面补个带网口能wol的笔记本替代，买了dell的OptiPlex3000 i5十二代准系统，待机功耗应该能到个位数，很适合后面改成linux机24小时开机，尤其dell、hp的机型。
  2. 一台低功耗软路由，不搞all in one那就每样都独立硬件，独立硬件那就要准求待机功耗了，毕竟n个硬件加起来每个5w也能十几二十w了。暂定r66s、r5s，arm可折腾性差点，但胜在功耗能到3w，而且lede维护，只能说精力财力妥协下的选择。
  3. Linux环境机，如果上面迷你主机用的好，可能直接捡垃圾再搞个迷你主机了。但买个战66感觉也行，5800u或者等6800u上市后这台带网口能自己配内存的机器成为我开发专用机，小尺寸也解决了目前游戏本略重的问题。
  4. mac mini，等m2上市后观望，最大用处当surge网关，但openclash足够胜任这活了，如果入手macmini前面的arm软路由大概就没啥用了 。又或者搞个m2 mba算了，但我好像实在没啥需求，虽然很想要个无噪音屏幕好又便携的机器。或者以后都用mba rdp到win开发机写代码，毕竟rdp实在好用又省流量。

进度

目前用一台十二代i5双网口机器带win server 2022，里面装ubuntu server、lstc win、ent win。

打算分成全局走软路由的机器和国内环境的机器。

国产流氓软件环境机

用了2周，原版lstc简单关闭一些服务后，开qq和微信，待机占用cpu保持在1%，目测待机多4-5w功耗。

接下来打算直接用精简的win10尝试是否能有效降低cpu占用。

Alpine Linux

部署了一台Alpine，用来做一些订阅转换等服务，使用体验上确实精简简单。

ROS

最近给ros配置了ipv6和防火墙，之前一直裸奔，网上找了好心人的手把手级别懒人教程，总算配了个ipv6防火墙。

主要是因为pt有些tracker之类连不上要靠ipv6，不得已重新启用ipv6，不知道会不会和openclash还冲突（以前开ipv6不适合用openclash）

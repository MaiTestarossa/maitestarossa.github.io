StackExchange.Redis timeout 问题研究

2021-07-07
CODE
约 467 字 预计阅读 1 分钟 次阅读

文章目录

  • .NET Framework 老项目的解决方案

.net下最常用的redis库StackExchange.Redistimeout问题似乎是个老生常谈的问题，不管是framework还是core都会遇到。

官方文档也专门有一个检查timeout问题的页面，虽然有一整页的说明，但也很难定位问题，几年来总有人断断续续问timeout相关问题。

在缺乏人力且没有精力深层研究redis相关性能的前提下，看上去坚持用se.redis不是好选择，尝试选用国内fork的csredis或者微服务化用其他语言跑redis相关操作或许是不错的治标不治本的捷径。

实在懒得改动太大的话，看社区很多人解决办法其中之一就是设置ThreadPool.SetMinThreads

.NET Framework 老项目的解决方案

官方文档说asp.net下应该改machine.config，但这样就会影响所有的.net项目了，对于老的framework项目还是尝试在代码中操作比较好。

经过实验，像console程序那样的方法，在asp.net中在Global.asmx的Application_Start同样有效果

1
2
3
4

        protected void Application_Start(object sender, EventArgs e)
        {
            ThreadPool.SetMinThreads(250, 250);
        }

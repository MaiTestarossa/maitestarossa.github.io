本文主要记录自己用的Firefox扩展推荐等。

我从Firefox 1.0a开始高强度使用Firefox。近几年基本是CentBrowser和Firefox混用，在CentBrowser逐渐降低更新频率以及对各种买办审核的忌惮，转而使用edge和chrome，现在是三个浏览器混用。

Firefox用于主力刷论坛社区，查资料。得益于Tree Style Tab相关生态，Firefox被我打造成了一个浏览器工作站一样的平台。

Chrome主要用于google差别对待浏览器的网站以及淘宝等网站。像Youtube之类网站都优先用Chrome观看，体验会比Firefox好；而淘宝等国内网站对pc端支持越来越差的环境下，很多时候只适配了Chrome，Firefox会有过不了反爬验证之类问题。

Edge用来管理本地nas、web服务等工作。因为Edge在国内用会自动转换成msn.cn相关内容提供，并且无法恢复（不管你是外国ip还是英文系统，无法转出去），调整到关闭所有新闻服务后新标签页只留下备案信息的狗皮膏药。虽然可以用新标签页扩展替代，对edge用起来还是有所忌惮，至少这流氓特性让我无法安心用。

uBlock Origin、暴力猴、油猴之类就不写了，新手推荐看更基础的推荐教程，本文假定用户有一定调教Firefox能力，只是作为补充。

扩展推荐

Tab Session Manager

会话管理扩展，以前用Session Sync，翻新的时候发现已经很久没更新了，且这个扩展是官方推荐。

这类扩展说实话用起来有点怕，指不定什么时候卖给什么数据公司开始偷偷上传你的会话记录。（以前用某个chrome官方商店下的扩展中过招，但不是会话相关扩展）既然是官方推荐，应该稍微有点保障。

Imagus

有个非常实用的功能，悬停显示原图，有时候如京东评论里面的买家晒图，只有靠这个扩展能轻松看见照片原图，比起看不到细节的小图，原图能看到商品更多细节，非常有帮助。

权限上有读取历史等权限，但有官方推荐标志，姑且相信他。

Disable WebRTC

随便找的禁用WebRTC扩展，注意uBlock自带的WebRTC功能不起作用 GitHub相关Issues，可以通过 https://browserleaks.com/webrtc 测试是否泄露了真实ip。

这里我老的Firefox配置却能在打开WebRTC的RTCPeerConnection和RTCDataChannel情况下检测不到Public Ip，试了改about:config也没效果，不知道以前自己怎么操作的。
